# 可见光通信协议规范 (v1.0)
**作者**: 霍环环  
**日期**: 2026-03-08

## 1. 概述
本协议定义了可见光通信系统的物理层参数，用于屏幕到摄像头的数据传输。

## 2. 物理层参数

| 参数 | 值 | 说明 |
| :--- | :--- | :--- |
| **屏幕分辨率** | 1920 × 1080 | 全高清 |
| **帧率** | 60 FPS | 视频播放帧率 |
| **数据区域** | 1008 × 1008 | 正方形数据区 |
| **块大小** | 18 × 18 像素 | 每个比特的显示区域 |
| **行列数** | 56 × 56 | 数据区网格划分 |
| **定位块大小** | 108 × 108 像素 | 四角定位标记 |
| **阈值** | 127 | 黑白判定阈值 (0-255) |

## 3. 帧结构

每帧总容量：**3136 bits** (56 × 56)

| 区域 | 占用格数 | 说明 |
| :--- | :--- | :--- |
| **定位锚点** | 144 (6×6×4) | 四角定位块，不存储数据 |
| **数据载荷** | 2976 | 有效用户数据 |
| **CRC校验** | 16 | CRC-16 校验位 |

**净数据率**: 2976 bits/帧 = 178,560 bits/秒 (60 FPS)

## 4. 定位锚点设计
四角各有一个 108×108 像素的定位块：
- 外框：白色填充 (255)
- 内框：黑色边框 (0)，宽度 10 像素
- 中心孔：黑色填充 (0)，大小 80×80 像素（右下角为 40×40）

## 5. 编码规则
- 比特 1 → 白色方块 (255)
- 比特 0 → 黑色方块 (0)
- 采样点：每个方块的中心像素

## 6. 校验算法
- **CRC-16**: 使用标准 CRC-32 的低 16 位
- 每帧独立计算和验证

## 7. 接口定义

### 发送端
```bash
python encoder.py <input.bin> <output.mp4> <duration_ms>
```
- `input.bin`: 输入二进制文件
- `output.mp4`: 输出视频文件
- `duration_ms`: 视频最大时长（毫秒）

### 接收端
```bash
python decoder.py <recorded.mp4> <output.bin> <vout.bin>
```
- `recorded.mp4`: 手机拍摄的视频
- `output.bin`: 解码输出的二进制文件
- `vout.bin`: 有效性标记文件

### vout.bin 格式
每字节表示 `output.bin` 中对应字节的8位掩码：
- 位为 **1**：表示该位正确接收
- 位为 **0**：表示该位错误（CRC校验失败）

例如：
- `output.bin`: `0x13 0x2F` (00010011 00101111)
- `vout.bin`:  `0xFF 0xFC` (11111111 11111100) 表示第二个字节的最后两位错误
