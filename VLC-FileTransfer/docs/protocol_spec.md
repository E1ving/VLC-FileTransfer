# 可见光通信协议规范 (v1.0)
**作者**: 霍环环  
**日期**: 2026-03-06

## 1. 概述
本协议定义了发送端（马驰）与接收端（王皓琳）之间的数据帧格式，旨在解决屏幕拍摄过程中的帧同步、丢帧检测及误码校验问题。

## 2. 帧结构设计
单帧总长度：**88 bits** (可配置)

| 字段名称 | 长度 (bits) | 说明 | 备注 |
| :--- | :--- | :--- | :--- |
| **同步码 (Sync)** | 8 | `10101010` | 用于接收端定位帧起始位置 |
| **序列号 (Seq)** | 8 | 0-255 循环递增 | 用于检测丢帧、重帧 |
| **数据载荷 (Payload)**| 64 | 有效用户数据 | 每次传输的核心比特流 |
| **校验码 (Checksum)** | 8 | CRC-8 / 异或校验 | 验证数据完整性 |

## 3. 设计决策与权衡
### 3.1 为什么选择 64 bit 载荷？
- **考量因素**：手机拍摄存在模糊和压缩噪声。
- **权衡**：
  - 载荷过长 -> 单帧出错概率增加，重传代价大。
  - 载荷过短 -> 头部开销（Sync+Seq+CRC）占比过大，有效传输率降低。
- **结论**：初步设定为 64 bit，后续根据误码率测试结果动态调整。

### 3.2 校验算法选择
- 采用 **8位异或校验** (初期) 或 **CRC-8** (后期优化)。
- **理由**：计算简单，适合实时视频流处理，且能检测出绝大多数单比特和多比特错误。

## 4. 接口定义
- **输入**: 原始二进制比特流 (`List[int]`)
- **输出**: 封装后的帧列表 (`List[List[int]]`)
- **依赖**: 发送端需按此顺序绘制黑白块；接收端需按此顺序解析。
